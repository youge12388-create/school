# 备份与恢复

## 需要备份的内容

必须同时保存：

1. `data/app.db`
2. `data/uploads/`
3. `data/keys/app.key`

只备份数据库无法恢复材料；只备份加密材料而丢失密钥也无法解密。

## 创建备份

建议先停止本地服务，再执行：

```powershell
npm run backup
```

脚本会先执行 SQLite WAL checkpoint，再将数据库、加密附件和密钥复制到：

```text
backups/YYYYMMDD-HHMMSS/
```

备份目录仍包含敏感资料，必须存放在启用 BitLocker 的磁盘或其他加密介质中。

## 恢复

1. 停止系统。
2. 将当前 `data/` 整体复制到隔离目录，不要直接删除。
3. 从同一个备份批次恢复 `app.db`、`uploads/` 和 `keys/app.key`。
4. 执行 `npm run db:migrate`。
5. 启动系统，抽查客户、申请和至少一个文件下载。

## 恢复验收

- 管理员可以登录。
- 学校和项目数量合理。
- 客户、申请和时间线完整。
- 随机下载至少三个不同客户的材料，确认均可解密。
- 审计日志仍可查询。

## 建议频率

- 试用期：每次批量导入或集中录入后备份。
- 正式使用：每日增量、每周完整备份。
- 每季度至少做一次恢复演练。
